# Sherpa-ONNX 语音识别 (ASR) WebUI 项目报告

## 1. 项目简介
本工程是一个基于 **WebAssembly (WASM)** 技术实现的网页端实时语音识别 (Automatic Speech Recognition, ASR) 系统。它使用 [sherpa-onnx](https://github.com/k2-fsa/sherpa-onnx) 引擎，在浏览器端直接运行 AI 模型，实现了无服务器依赖的本地语音转文字功能。

## 2. 核心架构与组件
项目由以下核心文件组成：
- **`index.html`**: 项目的 UI 界面，提供了录音控制、状态显示以及识别结果展示区域。
- **`app-asr.js`**: 核心业务逻辑，负责麦克风音频采集、采样率转换（下采样至 16kHz）、管理识别流以及处理 UI 交互。
- **`sherpa-onnx-asr.js`**: JavaScript 适配层，封装了与 WASM 引擎通信的接口。
- **`sherpa-onnx-wasm-main-asr.js/wasm`**: 编译后的核心推理引擎，执行高性能的深度学习模型推理。
- **`sherpa-onnx-wasm-main-asr.data`**: 包含预训练 ASR 模型權重和相关配置的资源包。

## 3. 主要功能特性

### 3.1 实时流式识别
- **低延迟**: 采用流式解码技术，用户在说话的同时，文字会近乎实时地出现在屏幕上。
- **端点检测 (VAD/Endpointing)**: 引擎能自动识别用户说话的停顿，并将识别出的句子分段记录。

### 3.2 多模态交互
- **手动控制**: 通过页面上的 `Start` 和 `Stop` 按钮控制录音。
- **快捷键支持**: 支持**空格键**长按功能（按下开始，松开停止），类对讲机体验。
- **自动滚动**: 识别出的文本会自动填充到 `Textarea` 中，并保持滚动条在最下方。

### 3.3 音频记录与回放
- **本地存储**: 每次录音结束后，系统会将采集到的音频数据实时转换为 **WAV 格式**。
- **历史记录**: 页面下方会生成音频剪辑列表，用户可以回放刚才的录音，并支持重命名或删除。

### 3.4 隐私与安全性
- **纯本地处理**: 所有音频数据均在用户的浏览器沙盒内处理，不上传至任何服务器，极大地保护了用户隐私。

## 4. 技术实现细节
1. **音频采集**: 使用 Web Audio API 获取麦克风输入的脉冲编码调制 (PCM) 数据。
2. **重采样**: 将浏览器默认的录音采样率（通常为 44.1k 或 48k）下采样为模型要求的 **16000Hz**。
3. **WASM 通信**: JavaScript 将音频 Buffer 传递给 C++ 编译的 WASM 模块，获取识别文本。
4. **WAV 封装**: 使用 `DataView` 在前端手动构建 WAV 文件头，实现 Blob 音频保存。
